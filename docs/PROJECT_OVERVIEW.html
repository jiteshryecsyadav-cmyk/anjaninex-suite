💾 Database Schema Organization

One PostgreSQL database, 8 logical schemas, Row-Level Security enforces firm isolation

platform.*
Firms, plans, wallet, changelog (Anjaninex)
core.*
Users, roles, contacts, branches
accounting.*
Heads, groups, ledgers, vouchers
trading.*
Parties, bills, payments, GR
suppliers.*
Supplier profiles, photos, rates
hr.*
Employees, attendance, payroll, GPS trails
ai.*
Extraction logs, cache
audit.*
Audit trail (immutable)

🚀 How to Run Locally

# 1. Open terminal at project root
cd "G:\Indian B2B SaaS platform"
# 2. Copy env template
copy .env.example .env
# 3. Start all infrastructure
docker compose up -d
# 4. Wait 30 sec, then seed data (run in order)
docker exec -i namokara_postgres psql -U namokara -d namokara_dev < db/seed/demo-data.sql
docker exec -i namokara_postgres psql -U namokara -d namokara_dev < db/seed/accounting-defaults.sql
docker exec -i namokara_postgres psql -U namokara -d namokara_dev < db/seed/trading-defaults.sql
docker exec -i namokara_postgres psql -U namokara -d namokara_dev < db/seed/suppliers-defaults.sql
docker exec -i namokara_postgres psql -U namokara -d namokara_dev < db/seed/hr-defaults.sql
# Terminal 1: Backend API
cd apps\api
dotnet restore
dotnet run
# → API available at http://localhost:5000/swagger
# Terminal 2: Frontend
cd apps\web
npm install
npm start
# → Web app at http://localhost:4200
